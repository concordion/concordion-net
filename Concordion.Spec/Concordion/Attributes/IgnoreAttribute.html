Ignore Attribute: Temporary omitting Tests

To allow you to include Concordion.NET specifications in the build before they have been fully implemented, you can annotate a fixture class with the NUnit IgnoreAttribute. When you add the [Ignore] attribute on top of your Concordion.NET fixture class, NUnit will not run this particular active specification. Instead, the test will be marked yellow and added to your test reports. This helps that the test will not be forgotten in future.

Example

When the fixture class associated with the [Ignore] attribute is passed to the NUnit runner

namespace Concordion.Spec.Concordion.Attributes
{
  [Ignore]
  [ConcordionTest]
  public class ExampleIgnoreTest
  {
    ...
  }
}

the test is not run.
